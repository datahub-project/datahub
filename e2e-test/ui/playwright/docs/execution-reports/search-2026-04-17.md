### Root Cause

The DataHub V2 entity page (`SearchPageV2`) no longer displays the platform name ("Hive") as visible text in the entity header. In the V2 redesign (`DefaultEntityHeader`), the platform is shown as an icon (`PlatformHeaderIcons`) only — the `PlatformContentView` with text is not used in the V2 layout. The `ContextPath` component shows the entity type but not the platform name.

### Fixes Applied

   - The remaining assertions (`expectTextVisible('Dataset')`, `fct_playwright_users_created`, `user_id`, `Id of the user`) are sufficient to verify successful navigation to the dataset page.

2. **`tests/search/query-and-filter-search.spec.ts:57-63`** - Replaced `expectTextVisible('Hive')` with `expectTextVisible('Dataset')` after clicking an entity result.
