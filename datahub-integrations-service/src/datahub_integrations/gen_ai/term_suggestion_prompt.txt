Task: Assign Appropriate Glossary Terms to Database Tables and Columns

This prompt guides you in assigning relevant glossary terms to tables and columns in a database. You will analyze provided table and column information, match it with appropriate glossary terms, and output the assignments in a specific format.

1. Input Information

1.1 Glossary Terms
A dictionary of available glossary terms. Each term has a unique identifier and contains:
- term_name: Name of the glossary term
- term_description: Description of the glossary term
- parent_node: Information about the parent category
  - name: Name of the parent node
  - description: Description of the parent node

Glossary Term Info: {glossary_terms_info}

Example:
{{
  "urn:li:glossaryTerm:PIITerms.GenderIdentity": {{
    "term_name": "GenderIdentity",
    "term_description": "The gender with which the user identifies.",
    "parent_node": {{
      "name": "Information Types",
      "description": "PII commonly found in data warehouses for a company creating design tools."
    }}
  }},
  ...
}}

1.2 Table Information
A dictionary describing the table:
- name: Name of the table
- description: Description of the table

Table Info : {table_info}

1.3 Column Information
A dictionary describing each column in the table:
- column_name: Name of the column
- description: Description of the column
- sample_values: Sample values of the column
- datatype: Data type of the column

Column Info : {column_info}

2. Output Format

Your output should be a JSON object with the following structure:
{{
  "table": [list of assigned glossary terms for the table],
  "column_name1": [list of assigned glossary terms for column1],
  "column_name2": [list of assigned glossary terms for column2],
  ...
}}

3. Confidence Score Guidelines
Assign a confidence score to each term-column match based on relevance, similarity, and substantiated reasoning. The higher the score, the more closely aligned the term is with the column. Use the following scale to determine the confidence score:

10: Perfect match
The term directly and unequivocally describes the column/table.
Example: "urn": "urn:li:glossaryTerm:customer_address" = "name": "Customer Address"
Rationale: The term "Customer Address" directly corresponds to the column's name, with no ambiguity or need for assumptions.

8–9: Strong match
The term is highly relevant and describes the column/table well, but might not be a perfect match.
Example: "urn": "urn:li:glossaryTerm:order_value" = "name": "Order Amount"
Rationale: "Order Value" closely matches "Order Amount", but there may be slight contextual differences (e.g., the type of currency or value unit), though the core concept is very similar.

5–7: Moderate match
The term is related to the column/table but requires assumptions to confirm the match.
Example: "urn": "urn:li:glossaryTerm:payment_method" = "name": "Payment Details"
Rationale: "Payment Method" is related to "Payment Details", but assumptions about what constitutes "details" (e.g., method, date, amount) need to be made to confirm the connection.

1–4: Weak match
The term is tangentially related or only loosely connected to the column/table. A match requires significant assumptions, and the connection is not well-supported.
Example: "urn": "urn:li:glossaryTerm:shipping_address" = "name": "Delivery Date"
Rationale: "Shipping Address" and "Delivery Date" are loosely related, as both pertain to shipping, but they are distinct concepts. The connection requires assumptions about whether the column relates to the timing of delivery, which isn't clearly indicated.


Each assigned glossary term should be an object with:
- urn: Unique identifier of the glossary term
- name: Name of the glossary term
- reasoning: Selection reasoning max 15 words + confidence reason max 20 words. Confidence reasoning is why did you decide to assign the score below instead of higher or lower
- confidence_score: Float from 1 to 10 (10 = perfect match). See confidence score guideline above before assigning.

4. Instructions

- When assigning terms, compare the term name and description with the column information to ensure a proper match.
- Assign only glossary terms from the provided list.
- Assign a maximum of 3 terms per column.
- Provide an empty list for columns with no matching terms.
- Do not suggest new terms or use terms not in the provided list.
- Use definitive language in reasoning; avoid words like 'likely' or 'maybe'.
- Ensure output is well-formatted and strictly adheres to the specified JSON structure.
- For partial matches or ambiguous cases, use a lower confidence score and explain the partial match in the reasoning.
- In case of overlapping terms, assign the most specific or relevant term and explain the choice in the reasoning.


Remember to adhere strictly to the provided glossary terms and format in your response.
